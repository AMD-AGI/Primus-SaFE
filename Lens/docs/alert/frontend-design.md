# Alert System Frontend Design

> Last Updated: 2026-01-19  
> Target Module: primus-lens-web

## 1. Overview

This document defines the frontend design for the Alert Management System, providing a comprehensive UI for multi-cluster alert rule configuration, alert event monitoring, and detailed alert analysis.

### 1.2 Design Principles

- **Multi-Cluster First**: All views support cluster filtering and cross-cluster operations
- **Unified Alert Model**: Consistent UI for metric/log/trace alerts
- **Real-time Updates**: Alert events auto-refresh with configurable intervals
- **Visual Severity**: Color-coded severity levels throughout (critical=red, high=orange, warning=yellow, info=blue)

### 10.3 API Service Example (alertEvents.ts)

```typescript
import request from '../request'
import type { AlertEvent, PaginatedResponse, AlertStatistics } from './types'

export interface ListAlertsParams {
  cluster?: string
  source?: string
  alertName?: string
  severity?: string
  status?: string
  workloadId?: string
  podName?: string
  nodeName?: string
  startsAfter?: string
  startsBefore?: string
  offset?: number
  limit?: number
}

export const alertEventsApi = {
  // List alerts with filters
  list(params: ListAlertsParams): Promise<PaginatedResponse<AlertEvent>> {
    return request.get('/alerts', { params })
  },

  // Get single alert detail
  get(id: string): Promise<AlertEvent> {
    return request.get(`/alerts/${id}`)
  },

  // Get alert correlations
  getCorrelations(id: string): Promise<AlertCorrelation[]> {
    return request.get(`/alerts/${id}/correlations`)
  },

  // Get alert statistics
  getStatistics(params: {
    dateFrom?: string
    dateTo?: string
    clusterName?: string
    groupBy?: 'hour' | 'day'
  }): Promise<AlertStatistics> {
    return request.get('/alerts/statistics', { params })
  }
}
```
